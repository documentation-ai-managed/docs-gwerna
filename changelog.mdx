## Upgrade Instructions

Follow these steps to upgrade Gwerna safely. Always test in a staging environment first.

<Steps>
  <Step title="Backup Your Data" icon="database">

    Export your documentation space:

````bash
gwerna export --space my-project --output backup.zip
````

  </Step>
  <Step title="Update CLI" icon="download">

    <CodeGroup tabs="npm,yarn">
````bash
npm install -g @gwerna/cli@latest
````
````bash
yarn global add @gwerna/cli@latest
````
    </CodeGroup>

  </Step>
  <Step title="Migrate Config" icon="settings">

    Update your `{config.json}` for new auth format:

````json
{
  "apiKey": "YOUR_API_KEY",
  "endpoint": "https://api.example.com/v1"
}
````

  </Step>
  <Step title="Verify Installation" icon="check-circle">

    Run `gwerna doctor` to check compatibility.

  </Step>
</Steps>

<Callout kind="alert">
  **Breaking Change in v1.1.0:** Authentication now uses `Authorization: Bearer {YOUR_API_KEY}` headers. Update all scripts immediately to avoid disruptions.
</Callout>

<Expandable title="Migration Example for v1.1.0" default-open="false">

If you used query param auth, here's a quick fix:

<CodeGroup tabs="JavaScript,Python">
````javascript
// Old
fetch('https://api.example.com/docs?api_key=YOUR_API_KEY')

// New
fetch('https://api.example.com/docs', {
  headers: { 'Authorization': `Bearer ${YOUR_API_KEY}` }
})
````
````python
import requests

# Old
response = requests.get('https://api.example.com/docs', params={'api_key': 'YOUR_API_KEY'})

# New
headers = {'Authorization': 'Bearer YOUR_API_KEY'}
response = requests.get('https://api.example.com/docs', headers=headers)
````
</CodeGroup>

</Expandable>
